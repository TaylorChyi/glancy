# 附录 B 合规模板与第三方清单

## B.1 目的与适用范围

- **目标**：为产品、法务、安全、采购与运营团队提供统一的合规评估模板，以及完整的第三方服务/子处理者登记表。
- **适用范围**：适用于所有面向最终用户的功能上线、版本迭代、第三方集成、采购及续约事项；包含实验性灰度与内测。
- **更新机制**：由产品负责人发起评估，合规/法务审批后存档；每季度例行回顾，或当法规、第三方状态发生变更时即时更新。

## B.2 合规评估模板（提交审批时填写）

| 模块 | 检查项 | 关键要点 | 责任角色 | 交付物/证据 |
| ---- | ------ | -------- | -------- | ------------ |
| 法规对标 | 适用法规识别 | 列出 GDPR、GDPL、《个保法》、COPPA、平台政策等适用条款；说明合规策略 | 合规负责人 | 法规适配矩阵（附件 B2-1） |
| 法规对标 | 数据跨境/本地化要求 | 标注数据驻留位置、跨境传输合法性依据 | 合规负责人 | 数据流向图、标准合同范本 |
| 数据生命周期 | 数据最小化与目的限制 | 声明收集字段、用途与必要性；是否存在二次用途 | 产品经理 | 数据收集清单、需求链接 |
| 数据生命周期 | 保留与删除策略 | 说明保留期限、逻辑删除与物理清理窗口 | 数据治理负责人 | 留存策略表、自动化脚本说明 |
| 用户权利 | 同意与撤回机制 | 描述告知、同意、撤回、选择退出流程 | 设计负责人 | UI 文案、截图、埋点策略 |
| 用户权利 | 数据访问与携带 | 提供导出格式、响应 SLA、身份验证流程 | 客服/运营 | 支持流程、工单模板 |
| 安全控制 | 访问控制与最小权限 | 说明角色、RBAC 配置、密钥管理 | 安全负责人 | 权限矩阵、密钥轮换记录 |
| 安全控制 | 加密与传输安全 | 标注传输协议、静态加密策略、密钥长度 | 技术负责人 | 架构图、配置截图 |
| 内部治理 | 第三方/供应商管理 | 是否新增或调整第三方；已完成尽调与合同 | 采购/法务 | 第三方评估报告、合同编号 |
| 内部治理 | 员工培训与审计 | 覆盖范围、频次、记录存证 | HR/合规 | 培训签到表、审计日志 |
| 风险管理 | 事中/事后应对计划 | 触发条件、应急联系人、通报路径 | 运营负责人 | 应急预案、联系人清单 |
| 风险管理 | DPIA/PIA 状态 | 判断是否需要开展，结论与补救措施 | 合规负责人 | DPIA 报告或豁免说明 |
| 审计追踪 | 指标与监控 | 定义监控指标、审计日志留存策略 | SRE/数据 | 监控看板链接、日志保留策略 |
| 审计追踪 | 发布与回滚记录 | 说明版本编号、灰度策略、回滚方案 | 发布经理 | 发布单、回滚预案 |

> **提交要求**：所有检查项需填写“状态（通过/风险/不适用）+说明”；风险项需附整改计划与预计完成时间。

## B.3 合同与合规审批流程

1. **发起**：产品/项目负责人提交《合规评估模板》与第三方变更单，附需求说明、数据流向图、界面稿。
2. **初审**：合规与法务完成法规对标、合同条款审阅，并出具“通过/整改/拒绝”意见。
3. **安全评估**：安全团队评估技术控制、漏洞扫描与渗透测试需求，必要时要求整改。
4. **管理层批示**：若涉及敏感个人信息或跨境传输，需升级到 DPO/数据安全委员会审批。
5. **存档与跟踪**：审批记录与附件归档至 Confluence/合同管理系统；监控整改任务按 OKR 跟踪。

## B.5 第三方管理与变更流程

1. **识别**：产品或技术团队提出新增/替换第三方需求，说明业务场景与数据类型。
2. **风险评估**：合规、安全、采购联合评估第三方安全、隐私、可用性、成本，形成评分卡。
3. **合同签署**：法务审核主合同、数据处理协议（DPA）、标准合同条款（SCC）或本地化备案。
4. **集成前检查**：技术团队确认访问控制、密钥管理、日志脱敏、开关控制与回滚机制。
5. **上线与监控**：发布后监控 SLA、费用、事件；按季度复审资质与审计报告。
6. **终止与迁移**：如终止合作，确保数据销毁证明、迁移计划与对用户的影响告知。
