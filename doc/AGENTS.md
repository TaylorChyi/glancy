# Agent 文档引用规范

## 背景

为保证知识库在多文件协作中保持一致且可追溯，任何 Agent 在撰写或更新 `doc/` 目录下的 Markdown 文档时，都必须确保读者能够直接跳转到被引用的章节或外部资源。因此需要统一引用方式，避免出现纯文本引用或不可点击的标题名。

## 基本要求

1. **必须使用超链接**：所有指向其他文档、章节、图示、Schema 或外部站点的引用都要使用 Markdown 链接语法 `[显示文本](目标路径或 URL)`。
2. **路径相对化**：引用仓库内的其他文件时，统一采用相对路径（例如 `./第 2 章 产品范围与目标.md#29-用户身份与模式定义`），确保在不同编辑器或发布渠道中均可正确跳转。
3. **章节锚点需精确**：引用同一文档内的具体段落时，附上锚点或显式行号，避免仅写章节标题。
4. **多处引用须逐条写明**：若一句话中提到多个来源，应分别提供对应超链接，而不是汇总成一个链接。

## 推荐格式示例

- ✅ `[第 2 章 2.9 用户身份与模式定义](./第 2 章 产品范围与目标.md#29-用户身份与模式定义)`
- ✅ `[附录 C 图示](./Appendix/附录 C 图示（系统上下文、时序、状态机）.md)`
- ✅ `[AC-UC-04 验收标准](./第 20 章 验收标准与测试方案（UC 对齐）.md#ac-uc-04)`

## 审核清单

- [ ] 文内是否存在裸标题或“参见某章”而无链接？
- [ ] 引用是否指向正确的文件/锚点并能在 CLI 与浏览器中打开？
- [ ] 外部 URL 是否可访问且使用 HTTPS？
- [ ] 模板、示例、图示等附属资源是否同样提供跳转入口？

如发现引用未提供超链接，视为阻塞问题，需在合并前修复。
