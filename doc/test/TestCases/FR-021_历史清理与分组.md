# FR-021 历史清理与分组（Must）

- **前置**
  - Plus 用户历史列表包含中英双语记录，至少 10 条 `zh`、10 条 `en`。
  - 授权可调用 `DELETE /api/v1/history` 与 `POST /api/v1/history/{receiptId}/restore`。
  - 服务器时钟与审计日志可查询以验证回执。
- **步骤**
  1. 进入 UC-05，选择“清理 zh 组”，在确认弹窗中勾选“我已知晓”并提交。
  2. 记录响应回执号 `receiptId`，观察页面提示耗时 ≤5 s。
  3. 立即刷新历史列表，确认 `zh` 组记录逻辑删除；`en` 记录保留。
  4. 在 T+3 天内调用恢复接口 `POST /history/{receiptId}/restore`，检查恢复结果。
  5. 待 7 天窗口后再次尝试恢复，验证失败提示。
- **期望**
  - 清理提交前强制二次确认，按钮默认禁用直至勾选。
  - 完成后返回 `receiptId`、范围（`lang=zh`）、操作人，UI 显示成功 toast（≤5 s）。
  - 恢复窗口内可按回执找回，恢复后状态写入审计；窗口过期返回 409 并指引用户。
- **数据**
  - API：`DELETE /api/v1/history?lang=zh`，`POST /api/v1/history/{receiptId}/restore`。
  - 审计：`history.cleanup`、`history.restore` 记录。
  - 词条样本：`词条 zh_*`、`term en_*`。
- **优先级**：P0（Must）
- **关联 UC**：UC-05
- **关联 AC**：AC-FR-021，AC-UC-05
